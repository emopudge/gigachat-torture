## Задание: ИИ-агент для генерации признаков

Вам нужно реализовать **ИИ-агента** на базе LLM [GigaChat-2-Max](https://developers.sber.ru/docs/ru/gigachat/models/gigachat-2-max), который автоматически работает с табличными данными:

- обрабатывает входные таблицы с помощью python-библиотек;
- понимает их структуру по описанию;
- генерирует признаки;
- оценивает их полезность на обучающей выборке;
- выбирает лучший набор признаков (**максимум 5 признаков**);
- сохраняет сгенерированные признаки для всех сэмплов датасетов `train` и `test`.

Главная цель задания — **показать, что ваш агент умеет самостоятельно придумывать и проверять хорошие признаки** для модели бустинга.

---

### Что важно понять сразу

В этом задании оценивается качество **фичей, которые сгенерирует агент**.

Именно поэтому:
- агент должен уметь работать автономно на универсальной задаче **бинарной классификации**;
- решение должно быть воспроизводимым;
- на проверке у организаторов будет использоваться **простая модель бустинга** на тех фичах, которые сгенерирует ваш агент (примерный скрипт скоринга можно посмотреть в `src/utils/scoring.py`);
- итоговое место в лидерборде зависит от того, насколько полезные признаки научится строить ваш агент.

---

## Формат соревнования

- Вам предоставлены датасеты для разработки и отладки вашего агента:
  1. [Датасет #1](https://disk.yandex.ru/d/_U2jv4D0L1RxBg)
  2. [Датасет #2](https://disk.yandex.ru/d/hnlkLp2--mA7Dw)
- Вы можете использовать дополнительные **открытые** датасеты, если это помогает сделать агента более устойчивым к разным типам табличных данных;
- После того как вы убедитесь, что агент корректно работает на предоставленных датасетах, нужно сделать submit на [платформе проверки решения](http://risk-hackathon.ru), загрузив zip-архив с кодом проекта;
- После сабмита ваш агент будет запущен на **скрытом датасете**;
- На проверке агент должен:
  1. прочитать данные;
  2. сгенерировать признаки для `train` и `test`;
  3. сохранить файлы `train.csv` и `test.csv` (в которых столбцы - ID записи, таргет (для `train`) и столбцы со сгенерированными признаками) в папку `output/`.
- После этого будет обучена **простая модель бустинга** на признаках, созданных вашим агентом, и будут получены предсказания на `test`;
- По качеству этих предсказаний будет сформирован лидерборд и определены участники следующего этапа.

Лимит сабмитов распространяется только на успешные попытки (**не более двух**). 

---

### Что именно нужно сдать

Вы сдаете **zip-архив с кодом**, который будет запущен на платформе проверки решений.

Ваш код должен:
- запускаться в чистом окружении;
- самостоятельно читать данные из папки `data/`;
- самостоятельно создавать признаки, проверять их и отбирать лучшие на `train`;
- сохранять результат в папку `output/`.

---

## Данные

Все входные данные находятся в папке `data/`.
Точный состав файлов зависит от конкретного датасета, но всегда описан в файле:
- `data/readme.txt` — **главный источник информации о данных**.

---

### Какие файлы находятся в `data/`

В папке `data/` будут файлы с таблицами в формате csv (разделитель может быть любым):

- `train.csv` — обучающая выборка;
- `test.csv` — тестовая выборка;
- дополнительные таблицы со справочниками, историей событий или другими связанными сущностями.

Так же:
- плоский файл `readme.txt` с описанием таблиц и задачи. 

---

### Важное про `train` и `test`

- В `train.csv` есть идентификатор объекта и целевая переменная;
- В `test.csv` - только идентификатор объекта;
- На финальной проверке будут использоваться признаки, созданные вашим агентом для обеих выборок.

---

## Что должен делать ИИ-агент

Ваш агент должен выполнять следующий пайплайн:

1. Прочитать данные из папки `data/`;
2. Прочитать и понять описание схемы из `data/readme.txt`;
3. Определить:
   - какие таблицы нужно использовать;
   - какие поля являются ключами;
   - как таблицы нужно объединять.
4. Обработать и проанализировать данные, сделать необходимые преобразования;
5. Придумать идеи генерации признаков;
6. Реализовать признаки и проверить их полезность;
7. Выбрать лучший набор признаков (**максимум 5**);
8. Построить итоговые признаки для `data/train.csv` и `data/test.csv`;
9. Сохранить результат в папку `output/train.csv` и `output/test.csv`.

---

### Что должно быть в `output/`

После запуска решения в папке `output/` обязательно должны появиться два файла:

- `train.csv` — файл с ID объектов, целевой переменной из обучающей выборки и сгенерированными признаками (максимум 5 штук) для каждого объекта выборки;
- `test.csv` — файл с ID объектов из тестовой выборки и сгенерированными признаками (максимум 5 штук) для каждого объекта выборки.

Набор признаков в `output/train.csv` и `output/test.csv` должен быть **одинаковым по структуре**:

- одинаковые названия колонок;
- одинаковая логика вычисления.

---

## Вариант генерации признаков

Агент может сделать несколько попыток генерации признаков.

Пример:
- в первой попытке агент придумал 5 признаков;
- во второй попытке агент придумал еще 5 признаков;
- далее агент сравнивает качество этих двух наборов;
- в финал идет **только один лучший набор**.

---

## Модель для проверки качества сгенерированных признаков

Для внутренней проверки качества признаков необходимо использовать **CatBoost**, именно на CatBoost с дефолтными значениями гиперпараметров (см. скрипт скоринга) будет проверяться результат работы агента на скрытом наборе данных.

---

## Требования к решению

- Окружение устанавливается командами:
```bash
uv venv
uv sync
```
- Все зависимости должны быть из `pyproject.toml`;
- Все входные данные читаются **только** из папки `data/`;
- Все выходные артефакты сохраняются **только** в папку `output/`;
- Код должен запускаться одной командой 
```bash
python run.py
```
- В `.env` нужно указать два параметра:
  ```bash
  GIGACHAT_CREDENTIALS='<Токен, полученный от организаторов>'
  GIGACHAT_SCOPE='GIGACHAT_API_CORP'
  ```
  `.env` в обязательном порядке должен быть в передаваемом zip-архиве;
- Решение не должно требовать ручного редактирования кода перед запуском;
- Решение не должно зависеть от локальных путей, настроек IDE или ручной подготовки окружения;
- Максимальное время работы агента **600 секунд**.

В файле `src/utils/check_submission.py` содержатся некоторые проверки соответствия Вашего решения указанным требованиям. 
После написания кода своего решения Вы можете проверить его работоспособность локально с помощью команды
```bash
python src/utils/check_submission.py
```
Перед запуском убедитесь, что в папке `data` размещены входные данные для задачи.
Делать сабмит на платформу имеет смысл лишь после прохождения всех проверок в данном файле.
